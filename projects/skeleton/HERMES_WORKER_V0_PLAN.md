# Hermes Worker v0 Plan

Status: public-safe planning note.

Hermes Worker v0 should be a controlled executor with a skill registry.

Phases:
1. Read-only preflight.
2. Public contract and schemas.
3. CLI dry-run skeleton.
4. Skill registry.
5. Controlled fallback.
6. First skill.
7. Install after approval.
8. Runner bridge.

Key rule: Hermes may propose skills, but cannot approve or activate them.
