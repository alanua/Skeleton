# Adaptive Practical Learning

Skeleton must improve with each step.

## Rule

When the same route fails, the next step must make execution better. Adaptation means practical progress, not extra paperwork.

## Required behavior

- After the first format failure, use the observed required format.
- After the first unknown allowlist item, check the real allowlist before creating another task.
- After the same blocker appears twice, stop creating similar tasks.
- Classify the repeated failure as ACTIVE_BLOCKER.
- Create the smallest repair task for the real blocker.
- Resume the original work only after the blocker is fixed or a safe workaround is proven.
- Record durable lessons in the correct project strategy or routing file after review.
